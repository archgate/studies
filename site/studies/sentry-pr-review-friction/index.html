Sentry PR Friction and ADR Standardization

Abstract. We analyze merged pull requests from the last 90 days in getsentry/sentry to quantify review friction and identify discussion themes that can be standardized into Architecture Decision Records (ADRs) and enforceable rules.

Method

  • Window: last 90 days (`merged:>=YYYY-MM-DD`, generated dynamically)
  • Source: GitHub API via `gh pr list`
  • Metrics: review events, merge latency, file count, churn, friction segmentation
  • Reproducibility kit: methodology code and outputs

Recurring Review Themes

  • API contract and compatibility concerns
  • Type/nullability and failure path semantics
  • UX and flow behavioral invariants
  • Test evidence expectations by change class
  • Permission and reliability guardrails

Proposed ADR Pack

ADR Intent Priority
